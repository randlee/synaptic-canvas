# Documentation Reference Notes

These notes provide the anchors referenced by `rust-development/guidelines.txt`.

## Documentation Has Canonical Sections (M-CANONICAL-DOCS) { #M-CANONICAL-DOCS }

Documentation should use predictable section shapes so readers and agents can locate purpose, inputs, outputs, operational constraints, and usage notes without reconstructing intent from scattered prose.

## Has Comprehensive Module Documentation (M-MODULE-DOCS) { #M-MODULE-DOCS }

Modules with real behavior should document responsibility, invariants, important dependencies, and correct caller usage. Do not leave significant modules as undocumented implementation buckets.
